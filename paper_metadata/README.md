# Paper Metadata Folder README

This folder contains the metadata and count summaries for the literature corpora used to generate the knowledge graphs (KGs).

## Folder contents

### 1. `Full_Paper_metadata.xlsx`
Original metadata file for the PFAS-specific full-text paper corpus.  
The Excel tabs are organized by publisher, since the full papers were collected by publisher.

- Total metadata entries: **4,825**
- Web of Science search results reported in the paper: **4,824**

### 2. `Full_paper_publisher_paper_counts.csv`
Summary file counting the number of full-text paper metadata entries from each publisher tab in `Full_Paper_metadata.xlsx`.

### 3. `Properties abstracts/`
Original property-abstract metadata files.  
The folder contains Excel spreadsheets organized by targeted material property keyword.

- Raw keyword-level abstract records across files: approximately **181k**
- After removing duplicates across property-keyword searches: approximately **160k unique abstracts**

### 4. `Abstracts_paper_counts.csv`
Summary file counting the number of abstract records in each property-keyword Excel file from `Properties abstracts/`.

### 5. `book_list.xlsx`
Metadata for the textbooks included in the manufacturing/materials-science RAG corpus (38 books, across `Full_list`, `Polymer books`, and `Additional` sheets). Each entry includes Title, Description, Publication Year, Publisher, eISBN, and — where available — DOI: 32 of the 38 books have a publisher-issued DOI, the remaining 6 have an eISBN only.

### 6. `corpus_DOI_inventory.xlsx`
Row-level inventory of the **entire** literature database, not just the subset used in the paper — combines `Full_Paper_metadata.xlsx` and `Properties abstracts/` into one sheet (`All_Records`, 186,360 rows). Each row is tagged:
- **Record Category** / **Text/Corpus Use**: `abstract only`, `full paper only`, or `abstract and full paper`
- **DOI Missing?**: flags rows with no DOI in the original source file
- **Used in Material Properties KG**: Yes/No, based on the 59,154 DOIs verified (see `kg_source_doi_report/` below) to have contributed an extracted relationship to `MatProp_62999.graphml`
- **Used in PFAS KG**: Yes/No — `PFAS_4612.graphml`'s edge-level source id is the source PDF's filename rather than a clean DOI, so this required active recovery, not just a direct match. 3,016 of the KG's 4,608 unique source identifiers (65.5%) normalize directly to a known DOI; the remaining 1,592 were run through CrossRef to recover the true DOI, of which 1,411 distinct DOIs (1,369 already in this inventory, 42 genuinely new) were verified and merged. See **`PFAS KG Match Method`** / **`PFAS KG Source Filename`** columns for provenance, and `pfas_missing_doi_recovery.xlsx` (below) for the full audit trail, including the 20 candidates excluded because they matched the wrong record (e.g. a "Correction to..." or "Comment on..." notice instead of the article itself) and the 135 left unresolved.

A `Summary` sheet gives the rollup counts behind these tags. Two data-quality corrections were applied during construction: 3 rows that were exact duplicates in the original `Full_Paper_metadata.xlsx` export (each counted twice) were deduplicated to one row each.

### 7. `corpus_summary.xlsx`
Standalone summary rollup of the same corpus statistics (record counts, missing-DOI counts, KG-usage counts, PFAS KG recovery methodology) for quick reference without opening the full row-level inventory.

### 8. `kg_source_doi_report/`
Reproducible, verified extraction of exactly which abstracts fed the Material Properties KG, generated by [`notebooks/extract_kg_source_dois.ipynb`](../notebooks/extract_kg_source_dois.ipynb). `matprop_used_dois_part{1,2}.json` list all 59,154 matched DOIs (100% match rate against `Properties abstracts/`) together with title, journal, year, and abstract text, split into two parts to stay under GitHub's 100MB per-file limit.

### 9. `pfas_missing_doi_recovery.xlsx`
Full audit trail for recovering DOIs of the 1,592 PFAS KG source identifiers that don't normalize directly to a known DOI. Records every candidate's resolution method (deterministic filename pattern vs. CrossRef bibliographic search), confidence score, and final disposition: merged into `corpus_DOI_inventory.xlsx`, excluded for matching the wrong record, or left unresolved.

## Note on the sampled ~60k abstracts
A random sample of **63,222 abstracts** was used for the Material Properties KG experiments due to the computational expense of graph merging, traversal, and retrieval at the full 160k-abstract scale. Independently verified against the KG itself (see `kg_source_doi_report/`): **59,154 unique DOIs** actually contributed an extracted relationship to `MatProp_62999.graphml`, all of which trace back to this folder's `Properties abstracts/`.
